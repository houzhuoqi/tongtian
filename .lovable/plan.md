## 现在为什么"丑"

逐条看了 `JiaoBeiThrow.tsx` + `styles.css` 里的 `bei-toss` / `bei-shadow`，主要问题：

1. **舞台是个干净的方框**，没有地面、没有透视，两只筊像漂在空气里。
2. **轨迹是镜像对称的"先飞高再落回"**：两只筊从中线左右各 44px 出发，又分别落到 ±46px，路径几乎平行，跟真实双手一起向前抛、落地后乱滚的感觉完全不同。
3. **"翻滚"只是 sprite 切帧 + Z 轴平面旋转**，没有近大远小、没有 Y 轴方向，所以看起来是"贴纸在抖"而不是"木块在翻"。
4. **着地节奏**：两只筊几乎同时弹跳两下后停住，落点没有横向滑动、没有第二次小翻面，像被磁铁吸住。
5. **静止姿态**也别扭：idle 时一只是 view-0（平面正视）、一只是 view-1（红凸 3/4），并排摆着像产品图，不像在桌上待抛。
6. **尘土 + 冲击环**触发得太对称、太亮，反而强化了"假"。

## 目标

让一次掷筊看起来像：双手合掌捧筊 → 朝供桌前方一抛 → 木块在空中真的翻 → 砸到桌面、滑一下、晃两下停住、扬起一点灰。

## 改造方案

### A. 加一个"供桌台面"作为舞台

容器里加一个透视台面（perspective + rotateX 的伪 3D 平面），暖棕色木纹渐变 + 顶部柔和受光、底部阴影。所有筊和阴影都画在这个平面上，立刻获得"地面"的参照。

```text
┌─────────────────────────┐
│        (空)              │   ← 抛起的空中
│                          │
│  ╲___桌面透视面___╱      │   ← rotateX(55deg) 的棕色面
└─────────────────────────┘
```

### B. 重做抛掷轨迹（核心）

不再用单一 `bei-toss` keyframes 控制两只筊。改成：

- **起手位置**：两只筊从画面下方"手部区域"（容器底部中央，间距很小，约 ±10px）一起被抛出。
- **空中段**：JS 计算每只筊一条略不同的抛物线（不同初速 vx/vy、不同落点），用 `requestAnimationFrame` 直接写 `transform: translate(x,y) scale(s) rotate(rz)`，不再走 CSS keyframes。这样可以：
  - 两只筊轨迹不再镜像对称
  - `scale` 跟随 `y`（近大远小）自然变化
  - 落点带一点随机散布（最终在 ±30~70px 内）
- **着地后**：再加一段 200~400ms 的"滑行 + 小翻"——`x` 继续向落点方向滑一点，并有 20% 概率多翻一帧（view-2/3 短暂闪过一次再回到结果面）。
- **静止**：定格在 `facesForResult` 给出的那一面，附带 ±15° 平面随机旋转和 ±4° 微微摇晃后收敛。

### C. 翻滚改用"轴向感"更强的帧调度

现在帧池是 `[view0, view2, view1, view3]` 循环切，速度由 `t/LAND_MS` 控制。改成：

- **由"剩余可用翻转次数"驱动**而不是固定时间间隔——基于落地总时长和目标终态，反推这次飞行需要 N 次翻面（4~7 次随机），均匀但带轻微 ease-out 分布在飞行段。
- **配合 scaleY 抖动**（0.92~1.08）让"边缘 → 平面"的瞬间像在过侧立，比纯切图更像 3D。
- **最后一帧锁定提前 ~80ms**完成，落地瞬间已经是结果面，不会出现"砸到地面才翻最后一下"的穿帮。

### D. idle 姿态改成"在手上的样子"

把两只筊改成在容器底部中央、轻微重叠、略向观者倾斜（一只压在另一只上 ~10px），并加一个非常缓慢的 4s 呼吸缩放（scale 0.99↔1.01），表示"被双手轻轻晃动"。"擲筊"按下后，先有一个 200ms 的"向上抬手"预备动作，再松手抛出——这一拍能极大提升真实感。

### E. 落地反馈降噪

- 尘土粒子数 14 → 6，颜色调到接近台面色，opacity 上限 0.35。
- 冲击环改成两段：第一段窄而快（落地瞬间），第二段稍宽淡出。
- 落地两只筊不再同时震动——错开 60~120ms，错位的节奏才像两块独立木头。
- `screen-shake` 幅度减半，只在两次都落地完毕后触发一次。

### F. 资源 & 代码组织

- 不动 `bei-real/view-*.png` 四张图。
- `JiaoBeiThrow.tsx`：
  - 抽出新组件 `BeiPhysicsToss`（rAF 物理小循环），替换现有 `BeiToss`。
  - `BeiStill` 改造为"待抛手部姿态"。
  - 删除现已不需要的 `--tx0 / --tx-land / --spin-from / --spin-to` CSS 变量驱动。
- `styles.css`：
  - 删除 `@keyframes bei-toss`（被 rAF 替代）。
  - 保留并微调 `bei-shadow`、`dust-puff`、`impact-ring`、`screen-shake`。
  - 新增 `.altar-surface` 透视台面样式。

### G. 验收

改完后我会自己跑预览，连抛 5 次截图，确认：

- 两次抛掷的轨迹/落点都不一样
- 空中翻转看起来像在翻木块而不是在切贴图
- 落地有"砸—滑—晃"三个动作
- 台面、阴影、灰尘色温统一，不再有刺眼的高光圈

## 不在范围

- 不改 `throwBei()` 概率逻辑、不改 `BeiResult` 类型
- 不替换贴图、不引入 Three.js
- 不改音效文件，只可能微调播放时机（与新落地时刻对齐）
